# 03. 트리 순회 (반복적 중위 순회)

재귀 없이 **명시적 스택**을 사용해 이진 트리를 중위 순회(inorder)합니다.

## 파일 구성
| 파일 | 역할 |
|---|---|
| `ItrTreeTrv.h` / `.c` | `TreeNode` 정의, 반복적 중위 순회 `itr_inorder_traversal` |
| `Stack.h` / `.c` | 순회에 쓰이는 연결 리스트 기반 스택 (`push`/`pop`/`is_empty`) |
| `main.c` | 트리를 만들고 순회 실행 |

## 알고리즘
중위 순회의 방문 순서는 **왼쪽 → 루트 → 오른쪽**. 재귀로 짜면 간단하지만,
여기서는 함수 호출 스택 대신 직접 만든 스택으로 그 동작을 흉내 냅니다.

```
현재 노드 p = root
반복:
  1) p가 NULL이 아니면 → 스택에 push 하고 p = p->left (왼쪽 끝까지 내려감)
  2) p가 NULL이면 → 스택에서 pop, 그 노드 방문(출력), p = 그 노드->right
  3) 스택이 비고 p도 NULL이면 종료
```

왼쪽으로 갈 수 있는 만큼 내려가며 스택에 쌓고, 더 못 가면 꺼내서 방문한 뒤
오른쪽으로 한 칸 이동하는 것이 핵심입니다.

## 왜 스택인가
재귀 = 시스템 콜 스택을 암묵적으로 사용하는 것. 그 콜 스택을 자료구조 스택으로
대체하면 같은 순회를 반복문으로 표현할 수 있습니다. 깊이가 매우 깊은 트리에서
재귀 한도(스택 오버플로) 문제를 피할 수 있다는 장점도 있습니다.

- 시간 복잡도: **O(n)** (모든 노드를 한 번씩 방문)
- 공간 복잡도: **O(h)** (h = 트리 높이, 스택에 최대 한 경로만큼 쌓임)
