# 08. 허프만 코딩 (Huffman Coding)

문자의 **출현 빈도**에 따라 가변 길이 코드를 부여하는 무손실 압축 알고리즘입니다.
자주 나오는 문자는 짧은 코드, 드문 문자는 긴 코드를 받아 전체 비트 수를 줄입니다.

## 핵심 구성 (`main.c`)
| 구성 | 역할 |
|---|---|
| `TreeNode` | 허프만 트리 노드 (문자, 빈도, 좌/우 자식) |
| `HeapType` + `insert_min_heap` / `delete_min_heap` | **최소 힙** (우선순위 큐) |
| `make_tree` | 두 노드를 합쳐 부모 노드 생성 |
| `GenerateHuffmanCodes` | 전체 절차 실행 |
| `printCodes` | 트리를 순회하며 각 문자의 코드 출력 |

## 알고리즘
**최소 힙을 우선순위 큐로** 사용하는 것이 핵심입니다. (빈도가 작을수록 우선순위 높음)

```
1) 모든 문자를 (빈도)를 키로 하는 노드로 만들어 최소 힙에 넣는다
2) 힙에 노드가 1개 남을 때까지 반복:
     a. 가장 작은 두 노드를 꺼낸다 (delete_min_heap ×2)
     b. 둘을 자식으로 갖고 빈도 = 두 빈도의 합인 부모를 만든다 (make_tree)
     c. 부모를 다시 힙에 넣는다 (insert_min_heap)
3) 마지막 남은 노드가 허프만 트리의 루트
```

## 코드 부여 (`printCodes`)
루트에서 각 잎(문자)까지 내려가며 **왼쪽=0, 오른쪽=1** 을 모읍니다.
잎에 도달하면 그동안 쌓은 비트열이 그 문자의 허프만 코드입니다.

- 빈도가 높은 문자일수록 루트에 가까워 코드가 짧아짐
- 어떤 코드도 다른 코드의 접두사가 아니어서(**접두 코드**) 디코딩이 모호하지 않음
- 트리 구성: **O(n log n)** (힙 연산 n번)

빈도가 작은 것끼리 계속 묶는 **그리디(greedy)** 전략이라는 점도 눈여겨볼 만합니다.
