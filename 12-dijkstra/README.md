# 12. 다익스트라 최단 경로 (Dijkstra)

한 시작 정점에서 **나머지 모든 정점까지의 최단 거리**를 구합니다.
가중치가 음수가 아닌 그래프에서 동작합니다.

## 핵심 구성 (`main.c`)
| 구성 | 역할 |
|---|---|
| `GraphType` + `EdgeNode` | 인접 리스트 그래프 |
| `MinHeap` + `insert_min_heap` / `delete_min_heap` | **우선순위 큐** (거리 기준 최소 힙) |
| `shortest_path` | 다익스트라 본체 |
| `print_status` | 각 정점까지의 현재 거리 출력 |

## 알고리즘
"아직 확정 안 된 정점 중 **가장 가까운 정점**을 확정"하는 그리디 방식입니다.

```
distance[시작] = 0, 나머지 = INF
최소 힙에 (시작, 0) 삽입
힙이 빌 때까지:
  현재까지 거리가 가장 작은 정점 u를 꺼낸다 (delete_min_heap)
  u가 이미 확정됐으면 건너뜀
  u를 확정하고, u의 인접 정점 v마다:
    if distance[u] + 가중치(u,v) < distance[v]:
        distance[v] 갱신       ← 더 짧은 경로 발견 (완화, relaxation)
        (v, 새 거리) 힙에 삽입
```

## 최소 힙이 핵심
매번 "가장 가까운 미확정 정점"을 찾아야 하는데, 이를 **우선순위 큐(최소 힙)** 로
처리하면 빠르게 꺼낼 수 있습니다. (`Found Order`로 확정 순서를 출력)

- 시간 복잡도: 힙 사용 시 **O(E log V)**
- **음수 가중치가 있으면 동작하지 않음** (한 번 확정한 정점을 다시 안 보기 때문)

핵심 아이디어인 **완화(relaxation)** — "더 짧은 길을 찾으면 거리를 줄인다" — 가
다익스트라의 전부라고 봐도 됩니다. MST(10·11)와 달리 **출발점 기준 거리**를
구한다는 점이 다릅니다.
