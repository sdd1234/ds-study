# 06. 힙 (최대 힙)

**최대 힙(max heap)** 을 구현합니다. 배열 기반 버전과 포인터 트리 기반 버전 두 가지를 담았습니다.

## 파일 구성
| 파일 | 역할 |
|---|---|
| `Heap.h` | `element`, `HeapType`(배열 기반) 구조체 정의 |
| `heap.c` | 배열 기반 최대 힙 + 대화형 인터페이스 |
| `main.c` | 실행 진입점 |
| `tree-version/main.c` | 같은 최대 힙을 포인터 트리로 구현한 버전 |

## 힙이란
**완전 이진 트리** 이면서, 모든 부모가 자식보다 크거나 같은(최대 힙) 구조.
루트에 항상 최댓값이 있어 우선순위 큐로 쓰입니다.

## 배열 기반 (`heap.c`)
완전 이진 트리는 빈틈이 없으므로 배열에 그대로 담을 수 있습니다 (1번 인덱스 시작).
- 노드 i의 부모 = `i/2`, 왼쪽 자식 = `2i`, 오른쪽 자식 = `2i+1`

| 연산 | 함수 | 동작 |
|---|---|---|
| 삽입 | `InsertMaxHeapTree` | 맨 끝에 넣고 **위로 올리기(up-heap)**: 부모보다 크면 교환 반복 |
| 삭제 | `delete_heap` | 루트(최댓값) 제거 → 마지막 원소를 루트로 옮기고 **아래로 내리기(down-heap)** |
| 출력 | `display_heap` | 레벨별로 출력 |

`runUserInterface`로 `i`(추가)/`d`(삭제)/`p`(출력)/`c`(종료) 메뉴를 돌려가며
힙이 어떻게 재정렬되는지(노드 이동 횟수 포함) 직접 관찰할 수 있습니다.

## 트리 기반 (`tree-version/`)
같은 최대 힙을 배열 대신 `TreeNode`(좌/우/부모 포인터)로 구현.
`heapifyUp`/`heapifyDown`으로 삽입·삭제 시 균형을 맞춥니다. 배열 인덱스 계산 대신
포인터로 부모·자식을 오가는 방식의 차이를 비교해 볼 수 있습니다.

- 삽입/삭제 모두 **O(log n)** (트리 높이만큼만 이동)

> 참고: `heap.c`는 제출 당시 원본 그대로라 헤더 선언과 일부 함수 시그니처가
