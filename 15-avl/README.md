# 15. AVL 트리 (균형 이진 탐색 트리)

스스로 균형을 맞추는 이진 탐색 트리입니다. 일반 BST의 약점(한쪽으로 치우치면
O(n)으로 느려짐)을 **회전(rotation)** 으로 해결합니다.

## 핵심 함수 (`main.c`)
| 함수 | 역할 |
|---|---|
| `getHeight` / `getBalance` | 노드의 높이, **균형 인수**(좌 높이 − 우 높이) |
| `rotateLeft` / `rotateRight` | 균형이 깨졌을 때 부분 트리 회전 |
| `insertNode` | 삽입 후 균형 인수를 보고 필요시 회전 |
| `deleteNode` | 삭제 후에도 동일하게 균형 유지 |
| `searchNode` | 일반 BST와 같은 방식의 탐색 |
| `doAVLBatch` / `doBinaryBatch` | AVL과 일반 이진 트리를 각각 2000번 시행해 비교 |

## 균형 유지 원리
모든 노드에서 **|좌 서브트리 높이 − 우 서브트리 높이| ≤ 1** 을 유지합니다.
삽입/삭제로 이 조건이 깨지면 회전으로 복구합니다. 깨지는 경우는 4가지:

| 경우 | 상황 | 해결 |
|---|---|---|
| LL | 왼쪽-왼쪽으로 치우침 | 오른쪽 회전 |
| RR | 오른쪽-오른쪽으로 치우침 | 왼쪽 회전 |
| LR | 왼쪽-오른쪽 | 왼쪽 회전 후 오른쪽 회전 |
| RL | 오른쪽-왼쪽 | 오른쪽 회전 후 왼쪽 회전 |

회전은 부분 트리의 부모-자식 포인터 몇 개만 바꿔 **O(1)** 에 끝나면서도
중위 순회 순서(=정렬 순서)는 그대로 보존합니다.

## 일반 트리와의 비교 실험
`doAVLBatch` / `doBinaryBatch`는 같은 데이터를 두 트리에 각각 넣고
탐색 시 **평균 비교 횟수**를 측정합니다.
- AVL: 항상 높이 O(log n) → 탐색도 O(log n)
- 일반 BST: 정렬된 데이터가 들어오면 한쪽으로 길어져 O(n)까지 악화

[`../05-binary-search-tree`](../05-binary-search-tree)의 BST가 왜 균형이 필요한지를
숫자로 보여 주는 마무리 단계입니다.
