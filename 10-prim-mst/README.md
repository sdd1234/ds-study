# 10. 최소 신장 트리 — 프림 (Prim's MST)

가중치 그래프에서 **모든 정점을 잇는 비용이 최소인 트리**(MST)를 구합니다.
프림 알고리즘은 **정점을 하나씩** 트리에 붙여 나가는 방식입니다.

## 핵심 구성 (`main.c`)
| 구성 | 역할 |
|---|---|
| `GraphType` | 인접 행렬 그래프 (`INF`로 간선 없음 표현) |
| `prim` | 프림 알고리즘 본체 |
| `GenerateGraph` | 테스트 그래프 생성·가중치 할당 |
| `FindPrimMST` | 실행 래퍼 |

## 알고리즘
하나의 트리를 시작 정점에서 **키워 나가는** 그리디 방식입니다.

```
시작 정점을 MST에 포함
정점이 모두 포함될 때까지 반복:
  MST에 속한 정점 ↔ 아직 안 속한 정점을 잇는 간선 중
  가장 가중치가 작은 간선을 고른다
  그 간선이 잇는 새 정점을 MST에 추가  (출력: "정점 N 추가")
```

`distance[]` 배열로 "각 외부 정점이 현재 트리에 붙는 최소 비용"을 관리하고,
정점이 추가될 때마다 인접 정점들의 `distance`를 갱신합니다.

## 특징
- 인접 행렬 + 선형 탐색 기준 **O(V²)** — 정점이 조밀한 그래프에 유리
- 항상 **연결된 하나의 트리**를 유지하며 키운다는 점이 크루스칼과 다름

## 크루스칼과 비교
같은 MST를 **간선 중심**으로 푸는 방식은 [`../11-find-mst`](../11-find-mst)에 있습니다.
- 프림: 정점을 하나씩 추가 (트리를 키움)
- 크루스칼: 가벼운 간선부터 추가 (숲을 합침)
