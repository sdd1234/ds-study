# 05. 이진 탐색 트리 + 스레드 트리

이진 탐색 트리(BST)를 만들고, 추가로 **스레드 이진 트리(Threaded Binary Tree)** 까지
구현해 순회 방식을 비교합니다.

## 파일 구성
| 파일 | 역할 |
|---|---|
| `bst.h` | `TreeNode`(일반), `ThreadNode`(스레드) 구조체 정의 |
| `bst.c` | 삽입, 순회, 스레드 트리 생성/순회 |
| `main.c` | 일반 트리와 스레드 트리의 중위 순회 결과 비교 출력 |

## 이진 탐색 트리 (BST)
각 노드는 **왼쪽 서브트리 < 자기 값 < 오른쪽 서브트리** 규칙을 만족합니다.

- `insert_node`: 루트부터 내려가며 키가 작으면 왼쪽, 크면 오른쪽으로 이동해 빈 자리에 삽입
- `inorder_traversal`: 중위 순회하면 **정렬된 순서**로 값이 나옴 (BST의 핵심 성질)
- 탐색/삽입 평균 **O(log n)**, 한쪽으로 치우치면 최악 **O(n)** → 이 한계가 [`../15-avl`](../15-avl)의 동기

## 스레드 이진 트리
일반 이진 트리는 잎 노드의 좌/우 포인터가 `NULL`로 낭비됩니다.
스레드 트리는 이 **빈 포인터를 중위 순회 상의 선행자/후속자(thread)** 로 재활용합니다.

- `GenerateThreadTree`: 일반 트리를 받아 빈 포인터에 후속 노드를 연결
- `ThreadTreeInOrder`: 스택이나 재귀 **없이**, 스레드만 따라가며 중위 순회

→ 두 방식의 중위 순회 출력이 같다는 것으로 스레드 연결이 올바름을 확인합니다.
스레드 트리의 장점은 **추가 스택 없이 O(1) 공간으로 순회**가 된다는 점입니다.
