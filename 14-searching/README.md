# 14. 탐색 알고리즘 비교

정렬된 배열에서 값을 찾는 여러 탐색 방법의 **평균 비교 횟수**를 측정·비교합니다.

| 폴더 | 비교 대상 |
|---|---|
| `01-compare` | 순차 탐색 vs 이진 탐색 |
| `02-interpolation` | 이진 탐색 vs 보간 탐색 |

## 순차 탐색 (`seq_search`)
앞에서부터 하나씩 비교. 정렬 안 돼 있어도 되지만 평균 **O(n)**.

## 이진 탐색 (`binary_search`)
정렬된 배열에서 **가운데와 비교** 후 절반을 버리는 방식.
```
low, high = 0, n-1
mid = (low + high) / 2
key < array[mid] → high = mid-1 (왼쪽 절반)
key > array[mid] → low  = mid+1 (오른쪽 절반)
key == array[mid] → 찾음
```
매 단계 탐색 범위가 절반 → **O(log n)**.

## 보간 탐색 (`interpol_search`)
이진 탐색의 변형. 가운데(mid) 대신, **찾는 값의 크기에 비례한 위치**를 추정합니다.
```
pos = low + (key - array[low]) × (high - low) / (array[high] - array[low])
```
값이 균등하게 분포돼 있으면 이진 탐색보다 빠를 수 있습니다.
- 균등 분포 시 평균 **O(log log n)**
- 분포가 치우치면 최악 **O(n)** 으로 나빠질 수 있음

## 정리
| 방법 | 전제 조건 | 평균 |
|---|---|---|
| 순차 | 없음 | O(n) |
| 이진 | 정렬됨 | O(log n) |
| 보간 | 정렬 + 균등 분포 | O(log log n) ~ O(n) |

각 `main.c`는 무작위 키를 수백~수천 번 탐색해 **평균 비교 횟수**를 출력합니다.
"정렬이라는 전제를 깔면 탐색이 얼마나 빨라지는가"를 수치로 체감하는 게 목표입니다.
