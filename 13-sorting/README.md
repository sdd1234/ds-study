# 13. 정렬 알고리즘 비교

여러 정렬 알고리즘을 직접 구현하고 **비교 횟수·이동 횟수**를 측정합니다.
폴더는 학습 순서대로 나뉩니다.

| 폴더 | 알고리즘 | 평균 시간 복잡도 |
|---|---|---|
| `01-basic` / `01-basic-v2` | 선택 정렬, 삽입 정렬 | O(n²) |
| `02-shell` | 셸 정렬 | O(n^1.3 ~ n²) |
| `03-merge` | 합병 정렬 | O(n log n) |
| `04-quick` | 퀵 정렬 | O(n log n) |

## 01. 선택 / 삽입 정렬 (기초)
- **선택 정렬 (`doSelectionSort`)**: 남은 구간에서 최솟값을 찾아 맨 앞과 교환. 항상 O(n²).
- **삽입 정렬 (`insertion_sort`)**: 앞에서부터 이미 정렬된 부분에 원소를 끼워 넣음.
  거의 정렬된 데이터에서는 매우 빠름(최선 O(n)).

`01-basic-v2`는 비교 횟수 측정 등을 다듬은 버전입니다.

## 02. 셸 정렬 (`gap_insertion_sort` / `doShellSort`)
삽입 정렬의 개선판. **gap(간격)** 만큼 떨어진 원소끼리 먼저 정렬하고,
gap을 점점 줄여(보통 절반씩) 마지막에 gap=1(=일반 삽입 정렬)로 마무리합니다.
멀리 있는 원소를 미리 큰 폭으로 옮겨 두어 삽입 정렬의 약점을 보완합니다.

## 03. 합병 정렬 (`merge` / `doMergeSort`)
**분할 정복**. 배열을 반으로 계속 쪼갠 뒤, 정렬된 두 부분을 `merge`로 합칩니다.
- 항상 O(n log n)으로 안정적, **안정 정렬**
- 단, 합치는 데 추가 배열이 필요 (O(n) 공간)

## 04. 퀵 정렬 (`partition` / `doQuickSort`)
**분할 정복**. `partition`으로 피벗(pivot)을 기준 삼아
작은 값은 왼쪽, 큰 값은 오른쪽으로 나눈 뒤 각 부분을 재귀 정렬합니다.
- 평균 O(n log n), 제자리 정렬로 실전에서 매우 빠름
- 피벗을 잘못 고르면 최악 O(n²)

## 정리
| 알고리즘 | 평균 | 최악 | 안정성 | 추가 공간 |
|---|---|---|---|---|
| 선택 | O(n²) | O(n²) | 불안정 | O(1) |
| 삽입 | O(n²) | O(n²) | 안정 | O(1) |
| 셸 | O(n^1.3~) | O(n²) | 불안정 | O(1) |
| 합병 | O(n log n) | O(n log n) | 안정 | O(n) |
| 퀵 | O(n log n) | O(n²) | 불안정 | O(log n) |

각 `main.c`는 같은 데이터를 여러 번 정렬해 **평균 비교 횟수**를 출력하도록 되어 있어,
이론상의 복잡도가 실제 연산량으로 어떻게 드러나는지 확인할 수 있습니다.
