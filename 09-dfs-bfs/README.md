# 09. 그래프 탐색 (DFS / BFS)

**인접 행렬**로 표현한 그래프를 두 가지 방식으로 탐색합니다.
깊이 우선(DFS)과 너비 우선(BFS)의 차이를 한 파일에서 비교합니다.

## 핵심 구성 (`main.c`)
| 구성 | 역할 |
|---|---|
| `GraphMatType` | 인접 행렬 그래프 (정점 수 + n×n 행렬) |
| `StackType` + `push`/`pop` | DFS용 스택 |
| `QueueType` + `enqueue`/`dequeue` | BFS용 큐 |
| `insert_vertex_mat_graph` / `insert_edge` | 정점·간선 추가 |

## DFS (깊이 우선 탐색) — 스택
한 방향으로 **갈 수 있는 데까지 깊이 파고든** 뒤 막히면 되돌아옵니다.

```
시작 정점을 스택에 push, 방문 표시
스택이 빌 때까지:
  v = pop, 출력
  v와 인접하고 아직 방문 안 한 정점을 모두 push
```
(재귀로도 동일하게 구현 가능 — 재귀의 콜 스택이 곧 이 스택)

## BFS (너비 우선 탐색) — 큐
시작점에서 **가까운 정점부터 레벨 단위로** 퍼져 나갑니다.

```
시작 정점을 enqueue, 방문 표시
큐가 빌 때까지:
  v = dequeue, 출력
  v와 인접하고 아직 방문 안 한 정점을 모두 enqueue
```

## 핵심 대비
| | DFS | BFS |
|---|---|---|
| 자료구조 | 스택(LIFO) | 큐(FIFO) |
| 탐색 모양 | 깊이 우선 | 거리(레벨) 우선 |
| 대표 용도 | 경로 존재 여부, 사이클 검출 | 최단 경로(간선 수 기준) |

- 인접 행렬 기준 시간 복잡도: **O(V²)** (각 정점마다 모든 정점과의 인접 여부 확인)
- 인접 리스트로 바꾸면 **O(V + E)** 로 개선 가능

자료구조(스택 vs 큐) 하나만 바꿔도 탐색 순서가 완전히 달라진다는 점이 핵심입니다.
