# 11. 최소 신장 트리 — 크루스칼 (Kruskal's MST)

MST를 **간선 중심**으로 구합니다. 가벼운 간선부터 차례로 골라,
사이클을 만들지 않으면 채택합니다. 정렬 방식 두 가지를 비교 구현했습니다.

## 핵심 구성 (`main.c`)
| 구성 | 역할 |
|---|---|
| `Edge` | 간선 (시작, 끝, 가중치) |
| `set_init` / `set_find` / `set_union` | **유니온-파인드(Union-Find)** — 사이클 판정 |
| `QuickKruskal` | `qsort`(퀵 정렬)로 간선 정렬 후 크루스칼 |
| `MinHeapKruskal` | **최소 힙**으로 간선을 하나씩 꺼내며 크루스칼 |
| `GenerateGraph` | 테스트 그래프 생성 |

## 알고리즘
```
모든 간선을 가중치 오름차순으로 처리:
  간선 (a, b)에 대해
  a와 b가 서로 다른 집합이면 (set_find로 확인)
    → 채택하고 두 집합을 합친다 (set_union)
  같은 집합이면 → 사이클이 생기므로 버린다
정점 수 - 1개의 간선을 채택하면 완료
```

## 유니온-파인드가 핵심
"이 간선을 넣으면 사이클이 생기나?"를 빠르게 판정하기 위한 자료구조입니다.
- `set_find`: 원소가 속한 집합의 대표(루트)를 찾음
- `set_union`: 두 집합을 하나로 합침
- 두 정점의 대표가 같으면 이미 연결돼 있다는 뜻 → 간선 추가 시 사이클

## 두 버전 비교
| | `QuickKruskal` | `MinHeapKruskal` |
|---|---|---|
| 간선 정렬 | `qsort`로 한 번에 전체 정렬 | 최소 힙에서 하나씩 꺼냄 |
| 장점 | 단순·구현 쉬움 | 일부만 필요할 때 유리 (전체 정렬 불필요) |

- 전체 시간 복잡도: **O(E log E)** (간선 정렬이 지배적)
- 간선이 적은(희소) 그래프에서는 프림보다 크루스칼이 유리한 경우가 많음

→ 정점 중심 방식인 프림은 [`../10-prim-mst`](../10-prim-mst) 참고.
